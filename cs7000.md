The [Connect Systems CS7000 M17 Plus](https://www.connectsystems.com/amateur-products/CS7000-M17-plus.html) is a radio that supports DMR and M17. The [CS7000 M17 PLUS CPS](https://www.connectsystems.com/amateur-software/CS7000_m17_plus_software.html) allows you to import digital contacts, channels, and zones from Excel spreadsheets. The Anytone to CS7000 code plug converter application converts a codeplug from the Anytone 878UVII Plus radio (CPS 3.04 through 4.0) into the spreadsheets that can be imported into the CS7000 CPS. The application is available as a stand-alone executable and is compatible with 64-bit Windows 10 and Windows 11 operating systems. Windows 7 is not supported.

## Features
- Supports codeplugs from Anytone 878UVII Plus (CPS 3.04 through 4.0)
- Provides option to load stock channels and zones that come pre-loaded in the CS7000 firmware (9.00.93)
- Provides option to load Contacts into the CS7000 using either the Table or Direct method. See Note 1
- Checks for duplicate Contacts and Channels and asks user to select which name to use. Duplicate names are remvoed. Contacts and Channels are updated accordingly.
- Removes any VHF channels from import as the CS7000 is a UHF only radio. In event a Zone only has VHF channels it is not imported.

Note 1: In the Table method the "TX Contact Name" field on the Channel is looked up in the Contacts database in the radio to obtain the ID (Talkgroup or DMR-ID). This is the standard method for provisioning Contacts into Channels. In the Direct method the "TX Contact Name" field is the ID (Talkgroup or DMR-ID) along with the "Contact Type" field to determine if the Contact is Private, Group, or All. The author is aware that the Direct method is available in firmware 9.00.93 and CPS 1.2.19.00BETA based upon correpondance with Jerry at Connect Systems. 

## Limitations
- Scan Lists are not imported. CS7000 CPS does not provide a way to import scan lists.
- Import of fields related to Squelch method  needs work. Guidance needed from users and Connect systems on how this should be implemented.
